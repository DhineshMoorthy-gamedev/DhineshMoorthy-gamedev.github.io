Unity Package Tracker

📅 March 2026 • ⏱️ 10 min read

Managing custom Unity packages shouldn't be a headache. The Unity Package Tracker is a premium Editor tool designed to track, manage, and safely remove local packages and custom .unitypackage imports with a modern UI Toolkit interface.

Why Unity Package Tracker?

Tracking what files were added by which package in a complex project is notoriously difficult. This tool solves that by providing:

  • 🚀 Modern UI Toolkit Interface — Pixel-perfect, dark-mode design with smooth transitions.
  • 🆔 GUID-Based Tracking — Survives folder renames and moves within your project.
  • 🔍 Smart Sidebar Filters — Audit all packages, recently added, or identify broken paths instantly.
  • 🌳 Interactive Asset Tree — Explore tracked assets hierarchically with "Click-to-Ping" integration.
  • 🛡️ Safe Physical Deletion — Cleanly uninstalls tracked assets while protecting source files.
💡 PRO TIP: The tool uses HashSet lookups to maintain O(1) performance even in projects with tens of thousands of assets.

Installation

Via Git URL

  1. Open Window → Package Manager.
  2. Click the + button → Add package from git URL...
  3. Paste: https://github.com/DhineshMoorthy-gamedev/Unity_PackageTracker.git

Manual Setup

  1. Clone the repository.
  2. In Package Manager, click + → Add package from disk...
  3. Select the package.json file.

Addressing the Problem

In standard Unity workflows, importing a .unitypackage is often a "one-way" operation. Once the assets are merged into your project, Unity doesn't inherently keep track of which file belongs to which package. This leads to several common frustrations for developers:

  • 📂 Cluttered Projects — It's difficult to identify and remove all files from a specific import once they've been scattered across different folders.
  • 🔗 Broken References — Manually deleting files often leads to missing scripts or broken prefab references if dependencies aren't tracked correctly.
  • 🕵️ Manual Auditing — Finding which package introduced a specific bug or asset requires tedious manual searching.

The Unity Package Tracker solves this by creating a reliable database of every file imported during a session. By using GUID-based tracking, it ensures that even if you move or rename those files later, the tool still knows exactly where they are and can safely remove them without leaving orphaned data behind.
